# Workflow：高管簡報（Executive Pitch / CEO 論壇）

> **適用場景**：EMBA 學員面對董事會、投資人、CEO 論壇、學校高管分享、產業研討會之簡報。
> **典型產出**：20–40 頁簡報 + 演講稿 + Q&A 準備。
> **時程**：構思到正式上台通常 2–4 週。

## 快速導讀

**高管簡報與學術簡報的差異**

| 面向 | 學術簡報 | 高管簡報 |
|---|---|---|
| 目標 | 傳遞資訊 | 驅動決策 |
| 結構 | 線性（問題→方法→結果） | 金字塔（結論→支持→細節） |
| 時間 | 20–40 分鐘 | 10–30 分鐘 |
| 視覺 | 表格、公式、引用 | 圖像、對比、故事 |
| 風險 | 方法有誤 | 無法打動決策者 |

**核心原則**
- 大老闆聽簡報不是要「完整」，是要「決策依據」
- 第一頁必須可獨立成立（不聽後續也知道你要什麼）
- 一張簡報一個重點，絕不塞滿

## Pipeline

```
Step 1：受眾與目標釐清
  問題：
    - 聽眾是誰？（董事會、投資人、同業 CEO、員工）
    - 他們已經知道什麼？
    - 你要他們做什麼決策／採取什麼行動？
    - 最壞情況他們反對的理由？
  skill：Asgard `meta-structured-problem`

Step 2：訊息骨架設計
  skill：
    - Asgard `hum-narrative`（敘事結構）
    - Asgard `hum-rhetoric`（說服技巧）
    - Asgard `meta-structured-problem`（金字塔原理）
  產出：一句話訊息 + 3 支持點 + 結構大綱

Step 3：內容填充
  skill：視主題選擇
    - Asgard `biz-swot`、`biz-porters-five-forces`（策略類）
    - Asgard `biz-pitch-deck`（pitch 基礎）
    - 本 repo `biz-innovation-management`（創新類）
    - 本 repo `biz-corporate-governance`（治理類）
    - 本 repo `biz-net-zero-transition`（ESG 類）
    - 本 repo `fin-m-and-a`（併購類）
  產出：每張簡報的具體內容

Step 4：視覺設計
  skill：Asgard `ops-pitch-deck`
  原則：
    - 每張簡報一個主角
    - 圖勝於表、表勝於字
    - 品牌一致性

Step 5：演講稿與演練
  skill：Asgard `hum-rhetoric`
  活動：
    - 完整演講稿（每張簡報 60–90 秒）
    - 至少 3 次完整演練（先獨自、再同儕、再目標聽眾代表）
    - 時間控制

Step 6：Q&A 準備
  skill：Asgard `hum-socratic`（預練反問）
  產出：20 個常見問題 + 回答要點
```

## 經典結構：SCQA Framework

```
S — Situation（情境）
    大家都同意的現況
    
C — Complication（複雜化）
    但是發生了什麼變化／問題
    
Q — Question（問題）
    所以我們該怎麼辦？
    
A — Answer（答案）
    推薦方案 + 三個支持理由
```

## 標準結構（30 分鐘高管簡報）

```markdown
【P1：封面】
  主題、副標題、日期、演講人

【P2：執行摘要（最關鍵）】
  ONE 結論 + 三個支持點 + 關鍵數字

【P3–5：情境建立（Situation）】
  - 市場／產業現況
  - 數據佐證
  - 為何現在討論這個議題

【P6–8：挑戰定義（Complication）】
  - 結構性挑戰
  - 競爭動態
  - 錯過的代價

【P9–15：分析與洞察】
  - 框架套用（1–2 個主要）
  - 數據分析
  - 關鍵洞察（bolded）

【P16–22：推薦方案】
  - 選項比較（通常 3 個：保守、基本、積極）
  - 推薦方案的邏輯
  - 財務估算（投資、回報、風險）
  - 實施路線圖

【P23–25：風險與應變】
  - 主要風險
  - 應變策略
  - 敏感度分析

【P26–28：結論與請求】
  - 回扣 P2 執行摘要
  - 明確的 ASK（我要董事會批准 / 撥款 / 決策）
  - 時程節點

【P29–30：Backup / Appendix】
  - 詳細數據
  - 敏感度分析
  - 備用方案
```

## 關鍵技巧

### 技巧 1：One-slide, one-message

```
❌ 壞例：
   標題：「台灣電子業的挑戰與機會」
   內容：10 個 bullet points

✅ 好例：
   標題（主張）：「我們有 18 個月窗口進入東南亞」
   內容：關鍵數據 1 + 圖表 1 + 結論 1
```

### 技巧 2：前 30 秒抓住注意力

**開場三選一**
- 驚人數據：「我們正站在過去 20 年最大的產業變動點」
- 對比衝擊：「同業 X 營收成長 40%，我們是 2%」
- 生動故事：「上週我拜訪客戶，他說...」

**避免**
- ❌ 「大家好，我是 XXX，今天要報告...」
- ❌ 「請看這張圖表...」（聽眾還沒準備好）

### 技巧 3：數字要有比較

```
❌ 「我們花了 2 億」
✅ 「我們花了 2 億，相當於去年淨利的 15%」
✅ 「我們花了 2 億，同業平均是 5 億」
```

### 技巧 4：故事 > 論證

**人類大腦對故事的記憶是論證的 22 倍**

- 用客戶故事代替統計
- 用員工故事代替組織圖
- 用競爭者故事代替 SWOT

### 技巧 5：視覺優先序

```
圖像 > 圖表 > 表格 > 文字
```

**圖表使用原則**
- 每張圖有一個「主角」（最重要的那條線／那個點）
- 顏色：主角用品牌色、其餘用灰
- 標題是結論（不是「圖 1：營收趨勢」而是「營收三年連續下滑」）

## Q&A 準備

### 常見高管提問類型

**類型 1：挑戰假設**
- 「如果你的市場成長率預估錯了呢？」
- **準備**：敏感度分析表、備案

**類型 2：挑戰執行力**
- 「我們有能力做這個嗎？」
- **準備**：關鍵人才、組織能力、外部夥伴

**類型 3：挑戰時機**
- 「為什麼是現在？」
- **準備**：窗口期、競爭動態、不做的代價

**類型 4：挑戰財務**
- 「ROI 多久回收？」
- **準備**：多情境（保守／基本／積極）、敏感度

**類型 5：挑戰風險**
- 「最壞情況？」
- **準備**：Worst-case plan、Stop-loss 條件

**類型 6：政治性提問**
- 「這跟去年 X 部門的專案衝突嗎？」
- **準備**：跨部門協作方案

### Q&A 的三種回答策略

**策略 1：直接回答**
- 確實知道答案 → 簡潔回答（< 30 秒）

**策略 2：分解問題**
- 問題複雜 → 拆成兩三個面向
- 「您問的是 X 嗎？我從 A、B 兩個角度回答」

**策略 3：承認不知道**
- 不知道 → **絕對不要編**
- 「我目前沒有這個數據，會後 24 小時內提供」
- 這比編答案更顯專業

## 演練清單

**基礎演練（全程錄影）**
- [ ] 時間控制（不超過規定 ±10%）
- [ ] 語速適當（中文 200–250 字／分）
- [ ] 眼神接觸（不低頭念稿）
- [ ] 手勢自然
- [ ] 停頓運用

**進階演練（有觀眾）**
- [ ] 互動自然
- [ ] 笑點掌握（若有）
- [ ] Q&A 處理
- [ ] 應對意外狀況（投影機壞、時間縮減）

**高管級演練**
- [ ] 找一位與真實聽眾相似背景者試講
- [ ] 記錄被挑戰的問題
- [ ] 修訂敏感點

## 不同情境的特殊要點

### 董事會報告
- 重視治理架構（本 repo `biz-corporate-governance`）
- 風險揭露完整（本 repo `biz-erm`）
- 財務嚴謹、符合會計準則
- 決議選項明確

### 投資人簡報（IPO、私募）
- 聚焦投資吸引力
- 財務預測透明（不過度樂觀）
- 退場機制清楚
- 風險完整揭露
- 合規避險（不得構成具體勸誘）

### CEO 論壇（校友活動、業界活動）
- 分享真實故事
- 學習啟發
- 不賣產品、賣洞見
- 展現個人領導風格

### 員工大會
- 簡化專業名詞
- 與員工利益連結
- 承諾明確
- 溝通管道留

### 記者會／媒體
- Message triangle（核心訊息三角）
- 不談細節（保留董事會決議）
- Bridging technique（帶回核心訊息）
- 對應本 repo 相關 skill（公關類用 Asgard）

## Gotchas

- **太多資訊**：聽眾記憶上限 3 點，不要塞 10 點
- **小字陷阱**：簡報字小於 18pt 後排看不見
- **動畫過度**：每張都飛進飛出 → 分散注意力
- **讀稿**：高管最討厭，寧可停頓思考
- **時間超時**：超過規定時間 = 不尊重
- **Q&A 爭辯**：聽眾說錯也不要強辯，「您的觀點很重要，我會帶回檢視」
- **技術故障**：Backup 檔案、印紙本、會場試過
- **文化忽視**：向日方簡報用西式結構 → 不合適；向董事長用新創風 → 不合適
