# Workflow：畢業論文（質性研究）

> **適用場景**：EMBA 畢業論文選擇質性方法（個案研究、紮根理論、行動研究、敘事分析、現象學）。
> **典型字數**：20,000–40,000 字（含 5–7 章標準結構）。
> **時程**：Proposal 通過到論文口試通常 6–12 個月。
> **產出**：完整學位論文，能通過校內口試並歸檔。

## 快速導讀

**EMBA 質性論文常見類型**
1. **單一個案深度研究**：一家公司的深度剖析（最常見）
2. **多個案比較**：2–4 家同業或跨業對比
3. **紮根理論**：從資料生成理論（較進階）
4. **行動研究**：學員親自介入 + 研究（合適自家公司）

**核心原則**
- 質性不等於「沒數字」，而是「深度重於廣度」
- EMBA 論文的優勢是「業界近身觀察」，必須發揮
- 學術誠信是底線（訪談同意、引用規範、資料保存）

## Pipeline

```
Phase 0：主題發想與可行性（1–2 月）
  活動：
    - 自己職涯 / 產業中的真實痛點
    - 文獻初探（是否已被做爛、是否有缺口）
    - 資料可及性（能否訪談到關鍵人）
  skill：
    - Asgard `meta-structured-problem`

Phase 1：Proposal 撰寫（2–3 月）
  活動：
    - 問題意識釐清
    - 文獻回顧
    - 研究方法選擇
    - 資料蒐集計畫
  skill：
    - Asgard `grad-systematic-review`（文獻回顧）
    - Asgard `grad-case-study`（個案研究）
    - Asgard `grad-grounded-theory`（紮根理論）
    - Asgard `grad-phenomenology`（現象學）
    - Asgard `grad-ethnography`（民族誌）
    - Asgard `grad-action-research`（行動研究）
    - Asgard `grad-narrative`（敘事）
    - Asgard `grad-paradigms`（研究典範）
  產出：5,000–10,000 字 Proposal

Phase 2：IRB 與同意書（1 月）
  活動：
    - 研究倫理審查（如校規要求）
    - 訪談同意書設計
    - 資料保存計畫
  skill：
    - Asgard `grad-survey-design` 部分適用
    - Asgard `law-gdpr-pdpa`（個資法）

Phase 3：資料蒐集（3–6 月）
  活動：
    - 訪談（深度訪談、焦點團體）
    - 觀察（參與式、非參與式）
    - 檔案（年報、內部文件、媒體報導）
    - 實地參訪
  skill：
    - Asgard `grad-case-study`（個案）
    - Asgard `grad-ethnography`（民族誌）
  產出：逐字稿、觀察筆記、文件清單

Phase 4：資料分析（2–4 月）
  活動：
    - 編碼（Open coding → Axial coding → Selective coding）
    - 主題分析（Thematic Analysis）
    - 敘事分析
    - 三角檢驗
  skill：
    - Asgard `grad-grounded-theory`（若用紮根）
    - Asgard `grad-narrative`（敘事分析）
    - 工具：NVivo、ATLAS.ti、MAXQDA、Dedoose

Phase 5：理論化與寫作（3–4 月）
  活動：
    - 結論形成
    - 與既有理論對話
    - 論文寫作
  skill：
    - Asgard `grad-systematic-review`
    - Asgard `hum-rhetoric`（論述寫作）

Phase 6：口試準備（1 月）
  活動：
    - 簡報製作
    - 問題預演
    - 修訂文稿
  skill：
    - Asgard `ops-pitch-deck`
    - Asgard `hum-socratic`（預練反問）
```

## 標準論文結構（EMBA 質性）

```markdown
第一章 緒論
  1.1 研究背景與動機
  1.2 研究目的與問題
  1.3 研究範圍與限制
  1.4 研究流程與章節安排

第二章 文獻回顧
  2.1 核心概念定義
  2.2 相關理論（主理論 + 次理論）
  2.3 實證研究回顧
  2.4 研究缺口與本研究定位

第三章 研究方法
  3.1 研究典範與方法論
  3.2 個案／樣本選擇邏輯
  3.3 資料蒐集方法
  3.4 資料分析方法
  3.5 研究信度與效度
  3.6 研究倫理

第四章 個案描述
  4.1 公司／現象背景
  4.2 重要事件時間軸
  4.3 關鍵利害關係人

第五章 發現與分析
  （按主題或時間軸展開）
  5.1 主題一
  5.2 主題二
  5.3 主題三
  （每主題含：證據引用、分析、理論對話）

第六章 討論
  6.1 研究發現總結
  6.2 與既有文獻對話
  6.3 理論貢獻
  6.4 實務意涵

第七章 結論
  7.1 研究結論
  7.2 研究限制
  7.3 未來研究建議

參考文獻（APA 格式）
附錄
  附錄 A 訪談大綱
  附錄 B 受訪者背景表（匿名）
  附錄 C 編碼範例
```

## 常見方法選擇指南

| 研究類型 | 適用情境 | 對應 Asgard skill |
|---|---|---|
| **單一深度個案** | 一家特殊企業（創新、危機、轉型） | `grad-case-study` |
| **多個案比較** | 同業異業對比、找共通 pattern | `grad-case-study` + `grad-systematic-review` |
| **紮根理論** | 新興現象、理論尚薄 | `grad-grounded-theory` |
| **行動研究** | 學員親自主導變革 | `grad-action-research` |
| **敘事分析** | 創辦人生涯、家族故事 | `grad-narrative` |
| **現象學** | 特定經驗的本質（如併購創傷） | `grad-phenomenology` |
| **民族誌** | 組織文化深度描繪 | `grad-ethnography` |
| **混合方法** | 質＋量並用 | `grad-mixed-methods` |

## 信度與效度檢核

**質性研究的四大信效度標準**（Lincoln & Guba）

1. **可信性（Credibility）**：三角檢驗、成員檢核、長期投入
2. **可轉移性（Transferability）**：豐厚描述、個案脈絡
3. **可靠性（Dependability）**：稽核軌跡、同儕討論
4. **可驗證性（Confirmability）**：研究者反身性、中立處理

**EMBA 論文常見檢核**
- 至少 8–15 位訪談（單一個案）或 10+ 位（多個案）
- 逐字稿完整保存
- 編碼過程可追溯（編碼簿、範例）
- 關鍵引言至少三人三方向佐證

## 學術誠信與倫理

**訪談前**
- 同意書（書面或錄音確認）
- 告知研究目的、資料使用、退出權利
- 匿名化處理（公司化名、職位概略）

**資料處理**
- 原始資料本人保存，不外流
- 引言加入「受訪者 A」等編碼
- 敏感資料（薪酬、個人隱私）雙層匿名

**AI 工具使用界線**
- ✅ 文獻搜尋、結構鷹架、中翻英（人工校對後）
- ✅ 編碼輔助（但需人工判讀最終）
- ❌ 代寫完整章節
- ❌ 偽造訪談資料（嚴重學術不端）
- ❌ 直接繳交 AI 初稿

**近年新規**
- 多校要求揭露 AI 使用
- 致謝章節或「研究方法」揭露工具
- 校方可能使用 AI 偵測軟體

## 寫作與時間節奏

**六個月密集型時程（參考）**

| 月份 | 主要任務 |
|---|---|
| M1 | Proposal 草擬、指導教授討論 |
| M2 | 文獻回顧深化、Proposal 通過 |
| M3 | 訪談對象聯繫、訪談大綱定稿、IRB |
| M4 | 密集訪談（目標 10 位以上） |
| M5 | 編碼與分析、章節骨架 |
| M6 | 完整寫作、口試準備、口試 |

**九個月標準型時程**（多個月緩衝）
- 建議多數 EMBA 學員採用
- 保留「業務生活衝擊」緩衝

## Gotchas

- **主題太大**：「台灣電子業的數位轉型」太大，「X 公司 CEO 主導 CRM 系統導入的組織學習歷程」才是論文
- **訪談不足**：至少 8 人、最好 12+，單一個案也是
- **資料迷戀**：收了 500 頁逐字稿但不知如何分析 → 編碼要在蒐集中段就開始
- **指導教授節奏**：每 2 週至少一次更新、不要神隱兩個月
- **理論對話不足**：EMBA 常犯「只描述個案，不跟文獻對話」
- **過度美化自家公司**：EMBA 論文常犯、會被口委挑
- **引用規範**：APA 第 7 版為主流、格式錯誤是基本分扣點
- **口試答辯練習**：指導教授 + 同儕至少模擬 2 次

## 指導教授搭配建議

**論文順不順，指導教授決定 70%**

- 選題領域相符
- 指導風格（放任 vs. 密集）與你節奏相合
- 過往指導學生論文品質（可調閱）
- 有無跨校共同指導資源
- 溝通管道是否順暢（EMBA 常需非上班時間）

**避開**：
- 超忙的明星教授（除非你能吸引注意）
- 與你產業完全無關的教授
- 被傳「卡畢業」的教授
