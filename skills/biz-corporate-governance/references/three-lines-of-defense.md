# 三道防線模型（Three Lines of Defense）詳解

## 起源與演進

- 1999 年巴塞爾委員會（Basel Committee）於銀行監理架構中首次提出類似概念
- 2013 年國際內部稽核協會（IIA）發布正式三道防線模型
- 2020 年 IIA 修訂為「**Three Lines Model**」（去掉 "Defense"，強調協作而非對抗）

## 模型架構（IIA 2020 修訂版）

```
┌─────────────────────────────────────────┐
│ 治理層（Governing Body / Board）           │
│   當責於利害關係人、對組織的監督             │
└─────────────────────────────────────────┘
         ↑                  ↑
   指導與監督            獨立保證
         ↓                  ↑
┌────────────────┐  ┌──────────────────┐
│ 管理階層        │  │ 第三道：獨立稽核     │
│  第一道 + 第二道│  │                     │
└────────────────┘  └──────────────────┘
         ↑
     外部保證（會計師、主管機關）
```

### 各角色職責

**治理層（董事會 + 審計委員會）**
- 當責於利害關係人
- 監督三道防線的獨立性與有效性
- 審計委員會是三道防線的直接對口

**第一道：業務單位（Business Operations）**
- 擁有與管理風險
- 執行控制流程
- 回報控制有效性

**第二道：風險管理、法遵、合規（Risk, Compliance）**
- 協助第一道設計控制
- 獨立監督第一道的風險管理
- 提供專業建議

**第三道：內部稽核（Internal Audit）**
- 獨立、客觀保證
- 評估三道防線的整體有效性
- 向治理層報告

## 2020 修訂版的關鍵變化

### 從「Defense」到「Model」
- **舊**：強調防範、守衛、靜態
- **新**：強調協作、動態、共同創造價值

### 從「線」到「角色」
- **舊**：三道防線像三條壕溝，涇渭分明
- **新**：三類角色可互動、支援，但保持獨立性

### 強化治理層角色
- **舊**：治理層在圖上方但較抽象
- **新**：明確治理層對三道的當責與監督職責

### 引入「外部保證」
- **舊**：外部會計師、主管機關未明確定位
- **新**：作為治理層的輔助監督

## 實務設計要點

### 第一道防線

**核心原則**：風險管理是所有人的責任，不是某部門的責任。

**實務工具**：
- 風險與控制自評（RCSA：Risk & Control Self-Assessment）
- 關鍵控制點（Key Controls）清單
- 日常監控報表
- 例外事件通報

**常見失靈**：
- 業務主管將控制外包給第二道 / 第三道
- 控制流於形式（紙上簽名、實際未執行）
- 異常發生後才建立控制

### 第二道防線

**核心原則**：**獨立但不取代**。第二道協助第一道，不是「幫第一道做事」。

**實務角色**：
- **風險管理部門（CRO / Risk Management）**：風險矩陣、壓力測試、風險胃納
- **法遵部門（Chief Compliance Officer）**：法規盤點、合規訓練、吹哨者
- **合規部門（Compliance）**：內控制度、政策 SOP
- **資訊安全（CISO）**：資安風險、資料治理

**組織歸屬**：
- 金融業：CRO / 法遵長向董事會報告（章程）
- 非金融業：可向 CEO 報告，但重大議題可繞過 CEO 向董事會
- **關鍵**：章程明定「獨立升級權」

**常見失靈**：
- 第二道直接參與業務決策（失去獨立性）
- 第二道人力過少，無法有效覆蓋
- 第二道主管由業務主管兼任

### 第三道防線

**核心原則**：**獨立性 + 專業**。內部稽核的獨立性是治理的最後一道防線。

**組織歸屬**：
- 功能性向 CEO 報告（行政、薪酬）
- 實質性向審計委員會報告（年度計畫、發現事項、稽核主管任免）

**稽核範疇**：
- 財務稽核
- 作業稽核
- 資訊稽核
- 合規稽核
- 績效稽核（近年增加）

**年度稽核計畫**：
- 以**風險導向**設計
- 高風險領域至少每年、中度每 2 年、低度每 3 年
- 計畫經審計委員會核准

**常見失靈**：
- 稽核主管由財會背景兼任、專業深度不足
- 稽核資源不足（人力、預算）
- 稽核報告被 CEO / CFO 過濾
- 稽核發現事項追蹤不力（重複發生）

## 升級機制（Escalation Mechanism）

### 三道之間的升級
- 第一道發現異常 → 立即通報第二道
- 第二道判斷重大 → 通報第三道 + 治理層
- 第三道發現重大缺陷 → 直接呈報審計委員會

### 各階段時限（建議）
| 事件類型 | 升級時限 |
|---|---|
| 一般控制缺陷 | 月度報告 |
| 重大控制缺陷 | 72 小時 |
| 重大舞弊跡象 | 24 小時 |
| 重大法律 / 監理事件 | 立即 |

### 升級失敗的常見原因
- 中層主管「捂蓋」避免升遷風險
- 第二道與被監督對象關係過於親密
- 第三道向 CEO 報告而非審計委員會
- 吹哨者制度未涵蓋高管

## 跨部門協作

### 三道共同議程的設計
- 每季「風險聯席會議」：三道主管共同出席
- 每年「風險地圖」更新：跨部門共創
- 重大專案「三道介入點」定義

### 避免「三道打架」
- 職責明確文件化
- 共同的風險分類語言
- 衝突處理升級機制

## 三道防線在不同產業的應用

### 銀行業
- 法規最完整（巴塞爾 III、金管會）
- 三道全部獨立部門、獨立預算
- 風控模型、資本適足率

### 保險業
- 類似銀行業但精算專業獨立
- 長期負債風險管理特殊

### 製造業
- 傳統以稽核為主、第二道薄弱
- 近年供應鏈風險、資安風險補第二道
- 海外子公司三道的延伸是挑戰

### 科技業
- 資安風險管理（CISO）為關鍵第二道
- 智慧財產、隱私、開源授權的合規
- 成長期常忽略、爆雷後補強

### 家族企業 / 未上市
- 三道常未完整
- IPO 前需補齊
- 成本壓力使小公司資源有限

## 三道防線與 ESG 的整合

### ESG 風險進入三道的機制
- **第一道**：各事業部的碳排放、勞工、供應鏈
- **第二道**：ESG 主管（CSO）、永續長
- **第三道**：ESG 稽核、永續報告保證

### 永續報告書保證（Sustainability Assurance）
- 獨立第三方保證（有限保證 / 合理保證）
- 逐漸成為法規要求
- 內部稽核可協助前置工作

## 海外子公司的三道防線

### 挑戰
- 總部三道無法實地監督
- 當地法規與文化差異
- 人事獨立性有限

### 設計原則
- **本地第一道**：當地業務主管
- **本地第二道**：當地法遵 / 風管（可由總部外派或當地招募）
- **本地第三道**：總部稽核每年實地 + 當地稽核人員（大型據點）
- **總部控制**：季度報告、年度檢討、稽核深度訪查

### 常見失敗
- 海外「獨立王國」 → 總部不敢深入
- 外派主管文化隔閡
- 當地法遵人員與業務主管「抱團」

## 三道防線的成熟度評估

### 成熟度五級
| 等級 | 特徵 |
|---|---|
| L1：初創 | 三道未明確、稽核偶爾做 |
| L2：發展 | 三道部署但部分兼任、文件化有限 |
| L3：標準 | 三道獨立、章程完整、定期報告 |
| L4：管理 | 三道有 KPI、升級機制順暢、跨部門協作 |
| L5：優化 | 三道整合、自動化工具、預測性風險管理 |

### 台灣企業分布（粗估）
- 上市金融業：多為 L4–L5
- 一般上市公司：多為 L3
- 非上市中大型：多為 L2
- 中小企業：多為 L1

## EMBA 個案分析建議

### 常見個案問題
1. 爆雷事件中三道為何失靈？
2. IPO 前如何 18–24 個月建立三道？
3. 海外子公司三道如何延伸？
4. 資源有限的中小企業如何務實設計三道？

### 診斷與建議結構
1. 每一道的**組織位置、資源、獨立性**盤點
2. 升級機制**時限、書面化、實測紀錄**
3. 近 3 年**重大控制缺陷**與處理
4. 與**同業 / 國際標竿**對照
5. 12–36 個月**改革路徑**
